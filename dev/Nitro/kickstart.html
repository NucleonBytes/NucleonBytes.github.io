NitroJS //Kickstart


Lets get your project on the road!


Getting started with NitroJS for the first time can be quite confusing. Specially if you dont stem from a web-development background. But don't worry! We have created this step-by-step guide to help you kickstart your project.

1. Setting up the manifest

To start off, we need to create a manifest file. This file will contain all the information NitroJS will need to launch your app. From there, NitroJS will take care of everything else. So create a new .jnx file and give it a name. For the purpose of this example, I will call mine texr.jnx (note how it has the jnx extension)


Once you have created the file, open it up and enter the following, filling in each parts appropriately



	{
		"manifest_version":3,
		"application":{
			"name":"texr",
			"version": "1.0.0",
			"homepage":"*/index.html",
			"icon":"*/star.ico",
			"description":"A small text editor",
			"permissions":["fs","app","system"]
		},
		"runtime":{
			"version":"1.0.0.6",
			"gpu_acceleration":true,
			"api":"v1.2"
		}
	}

Note how in this example, I have set the manifest_version to 3 and enabled gpu_acceleration. Remember that your description should not have a line break. Once you have correctly filled in the manifest file, save it!


2. Creating our first view

In NitroJS, we refer to webpages as views. Views are webpages that will be displayed in Nitro. In the previous step, we created our manifest file and set our homepage. The homepage is the first view that is processed by Nitro, so lets create that page now. To get started, create an HTML file (as you would normally do to create webpages) and give it the same name as the one you specified in your manifest for 'homepage'.



	<!DOCTYPE html>
	<html lang="en">
	<head>
		<meta charset="UTF-8">
		<title>texr</title>
	</head>
	<body>
		<style>
			pre{
				background: dimgray;
				color:white;
			}
		</style>
		<h1 id="myheading">Default heading</h1>
		<pre id="myeditor" contentEditable="true"> enter text here...	</pre>
		<input id="savebtn" onclick="saveit()" type="button" value="Save to desktop in file called texr-sample.txt">
		<script>
			var filepath = system.path("mydocuments") + "/texr-sample.txt";
			var username = app.get("currentUser"); //gets the name of the user currently logged in
			document.getElementById("myheading").innerHTML =username;
	
			function saveit(){
				var contents = document.getElementById("myeditor").innerHTML;
				fs.createFile(filepath);
				fs.writeFile(filepath,contents,false);
				alert("saved");
			}
		</script>
	</body>
	</html>

Anything you enter in between the 'title' tags will show up in your web-app's titlebar. Whereas anything you enter in the 'body' tag will be displayed as UI components in your app (just like in webpages)


In this example, we create a heading (which we will be changing to the name of the user currently logged in) and import jQuery for easy DOM manipulation


3. Run the web-app

Now that we have created this very simple web-app, it is ready for execution. There are two ways in which you can execute a .jnx file. First way is to drag the .jnx file over to the NitroJS.exe runtime. The second way is by right-clicking on the .jnx file and clicking 'open with...', then selecting NitroJS.exe
