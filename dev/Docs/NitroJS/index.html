var nitro = new NitroCore();
NitroCore is the core engine behind the multi-engine processor. Before you can start using the built-in components of the engine, you must create a new instance of the library.
nitro.addPermission('app');
When an instance of NitroCore has been initialized, you will need to grant access to your app - so you can query the app module for information about your app and its runtime.
app.get('appName');
Once you have been granted access to the app module, you may query it for information such as application name. (Note this is different from ApplicationName - which returns the namespace of your runtime)
myApplication
Note: Access to the App module is required!
app.close();
app.restart();
app.maximize();
app.minimize();
app.showBorder(boolean);
You can use the app module to set the properties of your application. For example, setting the border visibility.
Note: Access to the App module is required!
app.all();
The 'all' function returns a property list with all the information of your application.
Note: Access to the App module is required!
permission.add('permissionName');
The permissions module is provided with the Nitrix Runtime. In order to set and remove permissions without the NitroCore, you can use the permissions module directly. The permissions module is capable of granting access to one component at a time.
permission.remove('permissionName');
The permissions module is provided with the Nitrix Runtime. In order to set and remove permissions without the NitroCore, you can use the permissions module directly. The permissions module is capable of revoking access to a single component at a time.
permission.list();
The permissions module is provided with the Nitrix Runtime. It is equipped with a 'list' function which returns a list of all components with granted access.

system.run(optionsProperties);
System 'run' allows you to run programs installed on the client. It takes in options-properties such as 'verb' and 'appPath'.
Note: Access to the System module is required!
system.execute('pathName');
Once you have been granted access to the system module, you may invoke it to execute commands. The 'execute' function returns the exit code (if any)
01
Note: Access to the System module is required!
system.path('filepath');
The 'path' function returns a full file path value in return for the directory name (without spaces and no CAPS)
Note: Access to the System module is required!
